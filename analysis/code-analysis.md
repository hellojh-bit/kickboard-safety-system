# 코드 분석 정리

## 전체 개요

이 프로젝트는 전동 킥보드와 같은 개인형 이동장치의 안전한 이용을 위해 제작된 통합 안전 관리 시스템입니다.

5개 코드 파일은 각각 다음 역할을 담당합니다.

* Flask 서버 실행 및 API 처리
* SQLite 데이터베이스 생성 및 로그 저장
* YOLO 기반 헬멧 착용 여부 판별
* 라즈베리파이 기반 로드셀 하중 측정 및 모터 제어
* 웹 대시보드를 통한 실시간 하중 데이터 시각화

전체적으로는 사용자가 앱을 통해 킥보드를 연결하고, 헬멧 착용 인증을 진행한 뒤, 라즈베리파이가 로드셀 센서로 다인 탑승 여부를 판단합니다. 서버는 헬멧 인증 여부와 다인 탑승 여부를 종합하여 주행 허용 여부를 결정하고, 라즈베리파이는 그 결과에 따라 모터, LED, 경고음을 제어합니다.

---

## 파일별 분석

### 1. `server/app.py`

* 역할:
  Flask 기반 중앙 서버 파일입니다. 모바일 앱, 헬멧 판별 AI, SQLite DB, 라즈베리파이, 웹 대시보드를 연결하는 중심 역할을 합니다.

* 주요 기능:

  * Flask 앱 생성
  * CORS 설정
  * 업로드 폴더 생성
  * DB 초기화
  * QR 기반 킥보드 연결 처리
  * 회원가입 처리
  * 로그인 처리
  * 아이디 찾기 처리
  * 비밀번호 재설정 처리
  * 헬멧 이미지 업로드 처리
  * YOLO 기반 헬멧 착용 여부 판별 요청
  * 라즈베리파이 상태 수신
  * 헬멧 인증 여부와 다인 탑승 여부를 바탕으로 주행 허용 여부 판단
  * 웹 대시보드 페이지 제공
  * 최신 하중 데이터 API 제공

* 입력:

  * 모바일 앱에서 전달하는 사용자 정보
  * QR 연결 정보
  * 헬멧 인증 이미지 파일
  * 라즈베리파이에서 전송하는 8채널 로드셀 데이터
  * 브라우저의 대시보드 요청

* 처리:

  * 사용자 정보 확인 및 DB 저장
  * 업로드 이미지 저장
  * `AI_Helmet.py`의 `predict_helmet()` 호출
  * 헬멧 인증 결과 저장
  * 라즈베리파이에서 받은 무게 데이터 저장
  * `helmet_verified`와 `is_multiple_riders` 값을 기준으로 `is_unlocked` 판단

* 출력:

  * JSON 응답
  * 헬멧 인증 결과
  * 주행 허용 여부
  * DB 저장 결과
  * 대시보드 HTML 화면

* 비고:

  * 실제 파일명은 `main.txt`보다 `app.py`가 적절합니다.
  * `AI_Helmet.py`, `DB.py`와 같은 폴더에 있어야 import가 정상 작동합니다.
  * 실제 서비스에서는 비밀번호 암호화, 업로드 파일 검증, CORS 제한이 필요합니다.

### 2. `server/DB.py`

* 역할:
  SQLite 데이터베이스를 생성하고, 사용자 정보와 인증 로그, 하중 로그를 저장하는 DB 관리 파일입니다.

* 주요 기능:

  * `kickboard.db` 연결
  * 사용자 테이블 생성
  * 헬멧 인증 로그 테이블 생성
  * 로드셀 하중 로그 테이블 생성
  * 통합 주행 로그 테이블 생성
  * 헬멧 인증 결과 저장
  * 로드셀 하중 데이터 저장
  * 최신 하중 데이터 조회

* 입력:

  * 사용자 ID
  * 비밀번호
  * 이름
  * 전화번호
  * 헬멧 이미지 경로
  * 헬멧 판별 결과
  * 로드셀 8채널 무게값
  * 총 하중
  * 다인 탑승 여부

* 처리:

  * SQLite DB 연결
  * 테이블 생성
  * INSERT 쿼리 실행
  * UPDATE 쿼리 실행
  * SELECT 쿼리 실행

* 출력:

  * DB 파일 `kickboard.db`
  * 사용자 정보 저장 결과
  * 헬멧 인증 로그
  * 로드셀 하중 로그
  * 최신 하중 데이터 딕셔너리

* 비고:

  * 실제 파일명은 `DB.py`로 저장하는 것이 좋습니다.
  * 현재 비밀번호는 평문 저장 방식이므로 실제 서비스에서는 해시 처리가 필요합니다.
  * `kickboard.db` 파일은 개인정보가 들어갈 수 있으므로 GitHub에 올리지 않는 것이 좋습니다.

### 3. `server/AI_Helmet.py`

* 역할:
  YOLO 모델을 이용하여 업로드된 이미지에서 헬멧 착용 여부를 판별하는 AI 모듈입니다.

* 주요 기능:

  * `best.pt` 모델 파일 로드
  * 이미지 파일 분석
  * 감지된 객체의 클래스 ID 확인
  * class_id가 0이면 헬멧으로 판단
  * 헬멧 감지 여부를 `True` 또는 `False`로 반환

* 입력:

  * 이미지 파일 경로

* 처리:

  * YOLO 모델 실행
  * 감지 결과 반복 확인
  * 헬멧 클래스 존재 여부 판단

* 출력:

  * 헬멧 감지 시 `True`
  * 헬멧 미감지 시 `False`

* 비고:

  * 실제 파일명은 `AI_Helmet.py`로 저장하는 것이 좋습니다.
  * `best.pt` 모델 파일이 같은 폴더 또는 지정된 모델 폴더에 있어야 합니다.
  * 모델에서 class_id 0이 실제 헬멧 클래스인지 확인해야 합니다.
  * `ultralytics` 라이브러리가 필요합니다.

---

### 4. `raspberry-pi/rpi_algorithm.py`

* 역할:
  라즈베리파이에서 실행되는 하드웨어 제어 파일입니다. 로드셀 센서, DDSM115 모터, LED, 스위치, 경고음, 서버 통신을 담당합니다.

* 주요 기능:

  * 8채널 로드셀 데이터 읽기
  * 로드셀 영점 조절
  * 센서값 필터링
  * 총 하중 계산
  * 활성화된 로드셀 개수 계산
  * 하중 표준편차 계산
  * 다인 탑승 여부 판단
  * Flask 서버로 상태 전송
  * 서버 응답에 따른 주행 허용 여부 판단
  * DDSM115 좌우 모터 제어
  * LED 상태 표시
  * 다인 탑승 시 경고음 출력

* 입력:

  * 로드셀 센서값
  * 스위치 입력값
  * Flask 서버의 주행 허용 응답

* 처리:

  * 로드셀 원시값 읽기
  * 필터링 및 보정
  * kg 단위 무게 계산
  * 총 하중과 센서 분포 분석
  * 다인 탑승 조건 판별
  * 서버로 JSON 데이터 전송
  * 서버 응답에 따라 모터 ON/OFF 제어

* 출력:

  * 서버 전송 데이터
  * 콘솔 상태 출력
  * LED 색상 변경
  * 모터 구동 또는 정지
  * 경고음 재생

* 비고:

  * 실제 파일명은 `rpi_algorithm.py`가 적절합니다.
  * 일반 PC가 아니라 라즈베리파이 환경에서 실행해야 합니다.
  * `/dev/ttyUSB0`, `/dev/ttyUSB1` 포트 확인이 필요합니다.
  * 서버 IP 주소가 바뀌면 `SERVER_URL`을 수정해야 합니다.
  * 파일 마지막의 `//` 주석은 Python 문법 오류가 될 수 있으므로 `#` 주석으로 수정해야 합니다.

### 5. `server/templates/dashboard.html`

* 역할:
  실시간 하중 데이터를 시각화하는 웹 대시보드 화면입니다.

* 주요 기능:

  * 총 하중 변화 그래프 표시
  * 8개 로드셀 센서값 표시
  * 센서별 하중을 색상으로 표현하는 히트맵 표시
  * 다인 탑승 감지 시 경고 배지 표시
  * 1초마다 서버 API 호출
  * 최신 데이터로 화면 자동 갱신

* 입력:

  * `/api/realtime-weight` API 응답 데이터

* 처리:

  * JSON 데이터 파싱
  * 총 하중 표시 업데이트
  * 8개 센서값 업데이트
  * 센서값에 따른 색상 변경
  * Chart.js 그래프 데이터 갱신
  * 다인 탑승 여부에 따른 상태 배지 변경

* 출력:

  * 브라우저 화면에 실시간 그래프와 센서 히트맵 표시
  * 정상 탑승 또는 다인 탑승 경고 표시

* 비고:

  * 실제 파일명은 `dashboard.html`이 적절합니다.
  * Flask에서 사용하려면 `server/templates/dashboard.html` 위치에 있어야 합니다.
  * Tailwind CSS와 Chart.js를 CDN으로 불러오므로 인터넷 연결이 필요합니다.
  * 서버의 `/api/realtime-weight` API가 정상 동작해야 화면이 갱신됩니다.
