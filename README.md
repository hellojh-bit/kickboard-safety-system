# Kickboard Safety System

## 1. 프로젝트 소개

이 저장소는 **개인형 이동장치 통합 안전 제어 시스템** 프로젝트의 코드와 문서 자료를 정리한 저장소입니다.

본 프로젝트는 전동 킥보드와 같은 개인형 이동장치 이용 시 발생할 수 있는 **안전모 미착용**과 **다인 탑승** 문제를 사전에 감지하고, 위험 상황이 확인되면 주행을 제한하여 사용자의 안전을 확보하는 것을 목표로 합니다.

시스템은 크게 다음 요소로 구성됩니다.

* 모바일 앱을 통한 사용자 인증 및 헬멧 인증 요청
* Flask 서버를 통한 API 처리 및 데이터 관리
* YOLO 기반 안전모 착용 여부 판별
* SQLite 기반 사용자 및 주행 로그 저장
* 라즈베리파이 기반 로드셀 하중 분석
* 다인 탑승 감지 시 LED, 경고음, 모터 제어
* 웹 대시보드를 통한 실시간 하중 데이터 확인

## 2. 프로젝트 목적

개인형 이동장치의 보급이 확대되면서 안전모 미착용, 다인 탑승, 조향 불안정 등으로 인한 사고 위험이 증가하고 있습니다.

기존의 안전 관리는 사용자의 자율적인 준수나 사후 단속에 의존하는 경우가 많습니다. 본 프로젝트는 이러한 한계를 줄이기 위해, 주행 시작 전과 주행 중에 사용자의 안전 상태를 실시간으로 확인하고 위험 상황이 발생하면 물리적으로 주행을 제한하는 통합 안전 시스템을 구현하는 것을 목적으로 합니다.

특히 다음 두 가지 문제를 중점적으로 해결하고자 했습니다.

1. 안전모를 착용하지 않은 사용자의 주행 제한
2. 2인 이상 다인 탑승이 감지될 경우 주행 제한 및 경고

## 3. 주요 기능

### 1) 사용자 및 킥보드 연결

* 사용자 회원가입 및 로그인
* QR 코드 기반 킥보드 연결
* 킥보드 ID와 사용자 ID를 서버에서 세션으로 관리

### 2) AI 기반 헬멧 착용 판별

* 사용자가 업로드한 이미지를 Flask 서버로 전송
* YOLO 모델을 이용하여 헬멧 착용 여부 판별
* 헬멧 착용이 확인된 경우에만 다음 검증 단계로 진행

### 3) 로드셀 기반 다인 탑승 감지

* 라즈베리파이에 연결된 8채널 로드셀 데이터를 수집
* 총 하중, 활성화된 센서 개수, 하중 분포를 분석
* 다인 탑승으로 판단되면 모터를 정지하고 경고 상태로 전환

### 4) 주행 허용 및 차단 제어

* 서버는 헬멧 인증 여부와 다인 탑승 여부를 종합하여 주행 가능 여부를 판단
* 라즈베리파이는 서버 응답을 받아 모터, LED, 경고음을 제어
* 정상 상태에서는 주행 가능, 위험 상태에서는 주행 차단

### 5) 실시간 대시보드

* 최신 로드셀 데이터를 웹 화면에 표시
* 총 하중 변화 그래프 제공
* 8개 로드셀 센서 값을 히트맵 형태로 시각화
* 다인 탑승 감지 시 경고 표시

## 6. 실행 방법

이 저장소는 코드 및 자료 보관용 저장소입니다.
다만, 실제 실행하려면 서버 환경과 라즈베리파이 하드웨어 환경이 필요합니다.

### 1) Flask 서버 실행

Python 환경에서 필요한 라이브러리를 설치한 뒤 서버 파일을 실행합니다.

```bash
pip install flask flask-cors ultralytics
python server/app.py
```

서버 실행 후 기본적으로 다음 기능을 사용할 수 있습니다.

* 회원가입 API
* 로그인 API
* 헬멧 인증 API
* 라즈베리파이 상태 수신 API
* 실시간 대시보드 API

### 2) 대시보드 접속

Flask 서버 실행 후 브라우저에서 아래 주소로 접속합니다.

```text
http://서버IP:5000/dashboard
```

예시:

```text
http://10.42.0.1:5000/dashboard
```

### 3) 라즈베리파이 코드 실행

라즈베리파이에 로드셀, 모터, LED, 스위치가 연결되어 있어야 합니다.

```bash
python raspberry-pi/rpi_algorithm.py
```

라즈베리파이 코드 실행 전 확인해야 할 사항은 다음과 같습니다.

* 로드셀 8채널 연결 확인
* DDSM115 모터 컨트롤러 연결 확인
* `/dev/ttyUSB0`, `/dev/ttyUSB1` 포트 확인
* Flask 서버 IP 주소 확인
* 경고음 파일 경로 확인
* `lgpio`, `pygame`, `pyserial`, `requests` 설치 여부 확인

## 7. 코드 파일 설명

### `server/app.py`

Flask 기반 중앙 서버 파일입니다.
모바일 앱, 헬멧 판별 AI, SQLite DB, 라즈베리파이, 웹 대시보드를 연결하는 중심 역할을 합니다.

주요 기능:

* QR 기반 킥보드 연결
* 회원가입 / 로그인
* 아이디 찾기 / 비밀번호 재설정
* 헬멧 인증 이미지 업로드
* YOLO 모델을 통한 헬멧 착용 여부 판별
* 라즈베리파이 상태 수신
* 최종 주행 허용 여부 판단
* 실시간 대시보드 데이터 제공

### `server/DB.py`

SQLite 데이터베이스 관리 파일입니다.

주요 기능:

* 사용자 정보 테이블 생성
* 헬멧 인증 로그 저장
* 로드셀 하중 로그 저장
* 통합 주행 로그 저장
* 최신 하중 데이터 조회

### `server/AI_Helmet.py`

YOLO 기반 헬멧 판별 파일입니다.

주요 기능:

* `best.pt` 모델 로드
* 업로드된 이미지 분석
* 헬멧 객체 감지 여부 확인
* 헬멧 착용 시 `True`, 미착용 시 `False` 반환

### `server/templates/dashboard.html`

실시간 하중 관제 대시보드 화면입니다.

주요 기능:

* 총 하중 그래프 표시
* 8개 로드셀 센서값 표시
* 센서별 하중 히트맵 표시
* 다인 탑승 감지 시 경고 문구 표시
* 1초마다 서버 API를 호출하여 화면 갱신

### `raspberry-pi/rpi_algorithm.py`

라즈베리파이 기반 하드웨어 제어 파일입니다.

주요 기능:

* 8채널 로드셀 값 측정
* 총 하중 계산
* 활성화된 로드셀 개수 계산
* 하중 분포 및 표준편차 기반 다인 탑승 판단
* Flask 서버로 상태 전송
* 서버 응답에 따라 모터 제어
* LED 상태 표시
* 다인 탑승 시 경고음 재생
